# DIPractice


✔ DI(Dependency Injection)

핵심 개념: 사용할 객체를 클래스 내부에서 만들지 않고, 외부에서 주입받는 방식

객체 생성 위치: Program.cs 또는 IoC 컨테이너에서 생성

목적: 결합도 낮추기, 유연성 증가, Mock 객체로 테스트 쉬워짐

✔Factory

핵심 개념: 어떤 객체를 만들지 결정하고 생성 책임을 맡는 구조

객체 생성 위치: Factory 클래스 내부에서 생성하여 서비스에 전달

목적: 객체 생성 책임을 분리하여 코드 변경 시 영향 최소화


✔ Strategy

핵심 개념: 실행할 알고리즘(전략)을 외부에서 선택해 끼우는 구조

객체 생성 위치: Strategy 객체를 외부에서 생성 후 서비스에 주입

목적: 알고리즘/정책을 쉽게 갈아끼워 로직 변경을 유연하게 함
